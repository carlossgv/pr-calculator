## Current state
- The app compiles and the focused web test suite passes.
- The workout planner now behaves as a wizard, but the UX is still likely rough and may need further simplification.

## Decisions made
- Kept the workout flow inside the existing calculator experience rather than creating a separate route.
- Used a shared workspace component to avoid duplicating the tab logic between home and movement pages.
- Replaced the earlier optimizer-toggle model with a linear wizard so the user can progress in a more guided way.
- Kept the percent detail modal disabled in workout mode because it was not useful in that context.
